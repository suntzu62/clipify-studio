## Paul Graham — Startups & Founder Philosophy

**Fonte:** Essays paulgraham.com — "Startup = Growth", "Do Things That Don't Scale", "Default Alive or Default Dead"

### Princípios Fundamentais
- **Startup = Growth:** Uma startup não é simplesmente um negócio — é uma empresa desenhada para crescer rápido. Se não cresce, não é startup.
- **Growth rate define tudo:** 5-7% semana = excelente. 10% = excepcional. Abaixo de 1% = problema.
- **Do Things That Don't Scale PRIMEIRO:** Recrute manualmente os primeiros 10 usuários. Faça onboarding pessoal. Fique na cabeça dos early adopters. Só depois automatize.
- **Talk to users — obsessivamente:** O maior erro de fundadores é construir sem conversar com usuários. YC manda: "Make something people want" = descobrir o que querem CONVERSANDO.
- **Default Alive:** Receita atual - custos = fluxo. Se o fluxo for negativo e não tiver investimento, vc vai morrer. A pergunta é: "Se eu não levantar mais dinheiro, vou sobreviver?" Se não: é problema #1.
- **Ramen Profitability:** Ponto onde a empresa gera o suficiente para os fundadores sobreviverem. Meta mínima antes de qualquer escala.

### Para CortAI — Aplicação Direta
- **North Star:** Não é receita. É quantos criadores têm "aha moment" (primeiro clip viral gerado).
- **Do Things That Don't Scale:** Ofereça setup manual, ajude a escolher os melhores momentos, entregue valor antes do produto estar perfeito.
- **Conversas semanais:** Mínimo 3 entrevistas/semana com usuários enquanto <100 pagantes.
- **Default Alive calc:** MRR atual vs custos mensais (Render + OpenAI + Supabase + outros).

## Sam Altman — Founder Playbook (Y Combinator)

**Fonte:** "How to Start a Startup" (Stanford CS183B), Blog pessoal

### Princípios do Crescimento
- **Ideia → Produto → Time → Execução — nessa ordem:** No early stage, o produto e a ideia importam mais que o time. Contrate depois de ter tração.
- **Concentração extrema:** A maioria dos founders distribui energia demais. Foque em 1 métrica, 1 canal, 1 persona. Execute em profundidade.
- **Momentum is everything:** Pequenas vitórias sequenciais criam momentum. Momentum traz otimismo. Otimismo traz recrutamento e vendas. Quebre momentum = espiral descendente.
- **Be decisive:** Decisões ruins tomadas rápido > decisões perfeitas tomadas devagar. O custo de indecisão é subestimado.
- **Long-term thinking:** A maioria dos competidores age por 1-2 quarters. Quem age por décadas vence. Plante árvores cedo.

### Sobre Contratação (Para Solo Founder)
- Não contrate até ter PMF. Antes disso, você precisa de agilidade.
- Quando contratar: primeiro hire deve multiplicar sua capacidade, não apenas adicionar.
- Alternativa ao hire: automação + AI. O CortAI é uma empresa de AI — use AI internamente primeiro.

### Para CortAI — Aplicação Direta
- **1 métrica agora:** Número de clips exportados por usuário/semana. Essa é a métrica de ativação.
- **1 canal agora:** Qual canal gerou seus melhores clientes até hoje? Double down nele.
- **Decisão rápida:** Use o framework "70% de informação = suficiente para decidir".

## Ben Horowitz — The Hard Things About Hard Things

**Fonte:** Livro "The Hard Thing About Hard Things", Blog a16z

### Peacetime CEO vs Wartime CEO
- **Peacetime:** Empresa dominante no mercado, crescendo organicamente. Foco em cultura, processos, longo prazo.
- **Wartime:** Ameaça existencial — competidor, burn rate, churn explosive, falha de produto. Foco em sobrevivência, velocidade, decisões duras.
- **Solo founder early-stage = sempre Wartime.** Aplique o playbook de guerra.

### Wartime CEO Playbook
1. **Comunique o problema claramente** — sem suavizar. Você precisa de soluções, não de proteção emocional.
2. **Foque no problema mais crítico** — 1 coisa que, se resolvida, muda tudo.
3. **Execute em paralelos impossíveis** — Wartime CEO faz coisas que "não deveriam funcionar juntas".
4. **Não desperdice energia em consenso** — Tome decisões, comunique, execute.
5. **Coragem de demitir rápido** (no caso de SaaS: coragem de pivotar features rápido).

### Good Product Manager / Bad Product Manager
- Bom PM: define problema, entende usuário, prioriza brutal. Ruim PM: adiciona features por pressão.
- Aplique ao roadmap CortAI: CADA feature deve responder "isso retém usuário ou reduz churn?"

### Para CortAI — Aplicação Direta
- **Wartime agora:** Early-stage é sempre wartime. Cada decisão é potencialmente existencial.
- **Problema #1:** Identificar e focar no maior bloqueador de crescimento hoje (churn? ativação? aquisição?).

## Peter Thiel — Zero to One

**Fonte:** Livro "Zero to One: Notes on Startups"

### Monopoly Thinking
- Competição é para perdedores. Construa um monopólio.
- **4 pilares de monopólio:** Tecnologia proprietária (10x melhor) + Network Effects + Economies of Scale + Branding.
- Para CortAI: A tecnologia de IA de detecção de virality é o monopólio potencial. É 10x melhor que fazer manualmente?

### Segredo (Secret)
- Toda grande empresa foi construída em torno de um "segredo" — algo que você acredita que é verdade e o mundo ainda não sabe.
- **Segredo do CortAI:** "Criadores de conteúdo BR perdem 80% do valor dos seus vídeos por não reeditar em formato short. A IA pode desbloquear esse valor automaticamente."

### Last Mover Advantage
- Não importa ser o primeiro. Importa ser o ÚLTIMO — dominar o mercado no longo prazo.
- Como ser o último: Network effects que trancam usuários. Dados que melhoram o produto. Marca que significa confiança.

### Para CortAI — Aplicação Direta
- **Moat atual:** Dados de virality (quais clips performam bem) treinam o modelo e melhoram o produto. Dados = moat.
- **10x test:** Em qual dimensão o CortAI é 10x melhor que a alternativa (editar manualmente)? Tempo: manual = 2h, CortAI = 5 min. ✅

## Andrew Grove — OKRs (Intel, Google)

**Fonte:** "High Output Management", John Doerr "Measure What Matters"

### Framework OKR
```
Objective: Direção qualitativa inspiradora (O quê)
Key Results: Métricas mensuráveis que provam que você chegou lá (Como saberemos)

Regras:
- Máximo 3 Objectives por quarter
- Máximo 3-5 KRs por Objective
- KR = métrica + número + prazo
- OKRs são PÚBLICOS (mesmo para solo founder: escreva e revise semanalmente)
- 70% de atingimento = sucesso (100% = KR muito fácil)
```

### Para CortAI — Template OKR Q1 2026

O1: Atingir Product-Market Fit com criadores BR
  KR1: Must Have Survey ≥ 40% (atualmente: [medir])
  KR2: D30 Retention ≥ 35%

  KR1: 50 clientes pagantes ativos

## Reid Hoffman — Blitzscaling

**Fonte:** "Blitzscaling: The Lightning-Fast Path to Building Massively Valuable Companies"

### Quando Blitzscaling (AINDA NÃO para CortAI)
- Blitzscaling = crescimento a qualquer custo quando o mercado tem winner-takes-all dynamics
- **Pré-requisito:** PMF confirmado + mercado grande o suficiente + ameaça de competidor
- **CortAI agora:** NÃO blitzscale. Primeiro: PMF + Default Alive + Unit Economics positivos

### Network Effects (Para planejar)
- **Tipos de network effects:** Direto (mais usuários = mais valor para todos), Indireto, Dados
- **CortAI network effect de dados:** Cada clip gerado e exportado cria dados de virality que melhoram o ranqueamento. Isso é um data network effect — moat real.
- **Planejar para escala:** Quando PMF confirmado, o flywheel de dados acelera sozinho.

### Sinais de Alerta (CEO Alarmes)
- Churn >7%/mês = produto não está entregando valor consistente
- Ativação <20% (semana 1) = onboarding quebrado
- CAC payback >12 meses = modelo não sustentável
- Runway <6 meses = foco total em revenue ou corte de custos
- Must Have <30% = ainda não tem PMF, PARAR de escalar
